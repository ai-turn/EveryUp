# 알림 채널 설정 가이드

> English version: [NOTIFICATION_SETUP.md](./NOTIFICATION_SETUP.md)

EveryUp은 **텔레그램**, **디스코드**, **슬랙**을 알림 채널로 지원합니다. 이 가이드에서는 각 채널에 필요한 인증 정보를 발급받고 설정하는 방법을 안내합니다.

---

## 텔레그램 (Telegram)

### 1단계 — BotFather로 봇 만들기

1. 텔레그램에서 **@BotFather**를 검색합니다 (파란색 인증 마크 확인).
2. `/newbot` 명령어를 전송하여 봇 생성을 시작합니다.
3. 봇의 **표시 이름**을 입력합니다 (예: `EveryUp Alerts`).
4. 봇의 **사용자 이름(username)**을 입력합니다 — 반드시 `bot`으로 끝나야 합니다 (예: `everyup_alerts_bot`).
5. BotFather가 **봇 토큰(Bot Token)**을 응답합니다. 이 값을 복사해 두세요.

> **토큰 예시:** `123456789:ABCdefGHIjklMNOpqrSTUvwxYZ`

---

### 2단계 — Chat ID 확인하기

Chat ID는 EveryUp이 메시지를 어디로 보낼지 지정하는 값입니다. 개인 채팅, 그룹, 채널 모두 지원합니다.

#### 개인 채팅 (봇과 1:1)

1. 텔레그램에서 내 봇을 찾아 아무 메시지나 보냅니다 (예: `/start`).
2. 브라우저에서 아래 URL에 접속합니다:
   ```
   https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates
   ```
3. JSON 응답에서 `"chat":{"id": 123456789}` 를 찾습니다. `id` 값이 Chat ID입니다.

#### 그룹 채팅

1. 봇을 대상 그룹에 초대합니다.
2. 그룹에서 아무 메시지나 전송합니다.
3. 위의 `getUpdates` URL에 접속합니다.
4. `"chat":{"id": -100123456789}` 항목을 찾습니다. 그룹 Chat ID는 항상 `-100`으로 시작합니다.

#### 채널

1. 봇을 채널의 **관리자**로 추가합니다 ("메시지 게시" 권한 필요).
2. 채널에 메시지를 하나 전송합니다.
3. `getUpdates`를 확인하면 채널의 Chat ID가 표시됩니다 (`-100`으로 시작).

---

### 3단계 — EveryUp에서 설정하기

1. **알림 → 채널 추가 → 텔레그램**으로 이동합니다.
2. 표시 이름을 입력하고 **봇 토큰**과 **Chat ID**를 붙여넣습니다.
3. **저장**을 클릭하면 새 채널을 만들 때 연결 확인용 테스트 알림이 자동으로 전송됩니다.

---

### 문제 해결

| 증상 | 원인 및 해결 방법 |
|------|-----------------|
| `getUpdates`가 빈 `[]` 반환 | 봇이 아직 메시지를 받은 적이 없습니다. 봇에게 메시지를 먼저 보낸 후 다시 시도하세요. |
| 그룹에서 봇이 동작하지 않음 | 그룹 접근 권한을 활성화하세요: BotFather에서 `/mybots` → 봇 선택 → Bot Settings → Group Privacy → 끄기. |
| `401 Unauthorized` 오류 | 봇 토큰이 유효하지 않거나 만료되었습니다. BotFather에서 `/token`으로 새 토큰을 발급받으세요. |
| 테스트 알림이 수신되지 않음 | Chat ID를 다시 확인하세요. 그룹/채널의 경우 봇이 추가되어 있고 전송 권한이 있는지 확인하세요. 설정을 수정한 뒤에는 채널 카드의 **테스트** 버튼으로 다시 확인할 수 있습니다. |

## 디스코드 (Discord)

### 1단계 — 웹훅 만들기

1. 디스코드 서버에서 **서버 설정** → **연동** → **웹훅**으로 이동합니다.
2. **새 웹훅**을 클릭합니다.
3. 이름을 설정하고 (예: `EveryUp Alerts`) **대상 채널**을 선택합니다.
4. 원하는 경우 아바타 이미지를 업로드합니다.
5. **웹훅 URL 복사**를 클릭하여 URL을 저장합니다.

> **URL 예시:** `https://discord.com/api/webhooks/1234567890/ABCdefGHIjklMNOpqrSTUvwxYZ`

> **팁:** 서버 전체 관리자 권한 없이도 채널별로 웹훅을 만들 수 있습니다. 해당 채널의 **웹훅 관리** 권한만 있으면 됩니다.

---

### 2단계 — EveryUp에서 설정하기

1. **알림 → 채널 추가 → 디스코드**로 이동합니다.
2. 표시 이름을 입력하고 **웹훅 URL**을 붙여넣습니다.
3. **저장**을 클릭하면 새 채널을 만들 때 테스트 알림이 자동으로 전송됩니다.

---

### 문제 해결

| 증상 | 원인 및 해결 방법 |
|------|-----------------|
| `Invalid Webhook Token` 오류 | URL이 불완전하거나 잘못 붙여넣어졌습니다. `https://discord.com/api/webhooks/`로 시작하는 전체 URL을 사용하세요. |
| 테스트 알림이 수신되지 않음 | 웹훅이 삭제되었을 수 있습니다. 서버 설정 → 연동 → 웹훅에서 존재 여부를 확인하세요. |
| `404 Unknown Webhook` 오류 | 웹훅이 삭제되었습니다. 새 웹훅을 만들고 EveryUp에서 URL을 업데이트하세요. |
| `Webhook URL must use HTTPS` 또는 도메인 검증 오류 | EveryUp은 공식 Discord 도메인의 HTTPS 웹훅 URL만 허용합니다. |
| 전송 속도 제한(Rate limit) | 디스코드는 웹훅별 전송 횟수를 제한합니다. 잠시 후 자동으로 재시도됩니다. |

## 슬랙 (Slack)

### 1단계 — Slack 앱 만들기

1. [https://api.slack.com/apps](https://api.slack.com/apps) 에 접속하여 **Create New App**을 클릭합니다.
2. **From scratch**를 선택하고 앱 이름(예: `EveryUp Alerts`)을 입력한 뒤 워크스페이스를 선택합니다.
3. **Create App**을 클릭합니다.

---

### 2단계 — Incoming Webhooks 활성화

1. 앱 설정 페이지의 좌측 메뉴에서 **Incoming Webhooks**로 이동합니다.
2. **Activate Incoming Webhooks**를 **On**으로 전환합니다.
3. 페이지 하단의 **Add New Webhook to Workspace**를 클릭합니다.
4. 알림을 받을 **채널**을 선택하고 **허용(Allow)**을 클릭합니다.
5. 생성된 **Webhook URL**을 복사합니다.

> **URL 예시:** `https://hooks.slack.com/services/T.../B.../xxxx...`

> **팁:** 같은 Slack 앱에서 여러 웹훅을 만들 수 있으며, 각각 다른 채널로 알림을 보낼 수 있습니다. 최초 앱 설치 외 추가 권한은 필요하지 않습니다.

---

### 3단계 — EveryUp에서 설정하기

1. **알림 → 채널 추가 → 슬랙**으로 이동합니다.
2. 표시 이름을 입력하고 **웹훅 URL**을 붙여넣습니다.
3. **저장**을 클릭하면 새 채널을 만들 때 테스트 알림이 자동으로 전송됩니다.

---

### 문제 해결

| 증상 | 원인 및 해결 방법 |
|------|-----------------|
| `404` 또는 `invalid_payload` 오류 | 웹훅 URL이 유효하지 않거나 앱이 제거되었습니다. Slack 앱 설정에서 웹훅을 다시 생성하세요. |
| 테스트 알림이 수신되지 않음 | 봇이 채널에서 제거되지 않았는지 확인하세요. 채널의 **통합** 탭을 확인하세요. |
| `channel_not_found` 오류 | 대상 채널이 삭제되었거나 이름이 변경되었습니다. 올바른 채널을 지정하여 새 웹훅을 추가하세요. |
| `Webhook URL must use HTTPS` 또는 도메인 검증 오류 | EveryUp은 `hooks.slack.com`의 공식 HTTPS 웹훅 URL만 허용합니다. |
| 메시지 지연 | 트래픽이 많을 때 Slack이 메시지를 대기열에 넣을 수 있습니다. EveryUp이 자동으로 재시도합니다. |

---

## 공통 참고사항

- 새 채널을 저장하면 연결 확인을 위한 **테스트 알림**이 자동으로 전송됩니다. 기존 채널은 채널 카드의 **테스트** 버튼으로 언제든 다시 보낼 수 있습니다.
- 채널은 삭제하지 않고 **활성화/비활성화** 토글로 관리할 수 있습니다.
- 하나의 **알림 규칙**에 여러 채널을 동시에 지정하여 다중 채널로 알림을 전송할 수 있습니다.
- 알림 규칙에서는 조건(CPU 임계값, 오류율 등)과 알림을 받을 채널을 함께 설정합니다.
